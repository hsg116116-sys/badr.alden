## قائمة الأشياء **الناقصة** في المشروع

### 1. جداول SQL ناقصة من `supabase-setup.sql` (موجودة في Drizzle لكن مش موجودة في SQL)

| الجدول | المكان |
|--------|--------|
| `pos_sessions` | `lib/db/src/schema/pos.ts:5` |
| `pos_orders` | `lib/db/src/schema/pos.ts:16` |

هذين الجدولين معرفين في Drizzle ORM لكن مش مضافين في ملف SQL setup.

### 2. جداول SQL موجودة في setup.sql لكن **ناقصة من Drizzle Schema**

| الجدول | مكانها في SQL |
|--------|---------------|
| `discounts` | `supabase-setup.sql:197` |
| `counters` | `supabase-setup.sql:217` |
| `settings` | `supabase-setup.sql:225` |
| `journal_entries` | `supabase-setup.sql:161` |
| دالة `increment_counter()` | `supabase-setup.sql:231` |

ما في Drizzle schema files لهذه الجداول — فما تقدر تستخدم Drizzle ORM معاها.

### 3. مشاكل عامة في قاعدة البيانات

- **لا يوجد Indexes** — مافي `CREATE INDEX` على أي عمود (مثل `sku`, `barcode`, `invoiceNumber`, `phone`)
- **لا يوجد RLS Policies** — مافي `ALTER TABLE ... ENABLE ROW LEVEL SECURITY` — قاعدة البيانات مفتوحة لكل مستخدم
- **لا يوجد Foreign Keys للتقييد** — `discounts.bulkProductId` و `discounts.categoryId` معرفين كـ INTEGER بدون `REFERENCES`
- **لا يوجد User/Auth System** — مافي جدول `users` أو `profiles` مرتبط بـ Supabase Auth لتحديد المستخدمين وصلاحياتهم
- **أنواع TEXT للتواريخ** — `createdAt`, `date` مخزنة كنص (TEXT) بدل `TIMESTAMPTZ` — ما تقدر تعمل sorting صحيح أو queries زمنية
- **أنواع TEXT للأرقام** — `unitPrice`, `costPrice`, `stock`, `amount`, `salary` مخزنة كنص (TEXT) بدل `NUMERIC` — مخاطرة لعمليات الحساب

### 4. إعدادات البيئة الناقصة

`.env` يحتوي فقط على Supabase keys:
```
NEXT_PUBLIC_SUPABASE_URL=...
NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY=...
VITE_SUPABASE_URL=...
VITE_SUPABASE_PUBLISHABLE_KEY=...
```

**ناقص:** `DATABASE_URL` (مطلوب لـ `lib/db/src/index.ts` و `drizzle.config.ts`) و `PORT` (مطلوب لـ API server)

### 5. API Routes موجودة لكن بعضها بدون SQL tables كاملة

- **`routes/pos.ts`** — يستخدم `pos_sessions` و `pos_orders` (غير موجودين في SQL setup)
- **`routes/dashboard.ts`** — يحتاج تجميع بيانات عبر جداول متعددة
- **`routes/reports.ts`** — يحتاج queries تحليلية متقدمة

### 6. أجزاء الفرونت أند بدون Backend API

- **`Pages/Discounts.tsx`** — صفحة موجودة لكن مافي API route للـ `discounts`
- **`Pages/InvoiceSettings.tsx`** — بيستخدم `settings` table (موجود في SQL لكن مش في Drizzle)
- **`Pages/Journal.tsx`** — بيستخدم `journal_entries` table (موجود في SQL لكن مش في Drizzle)
- **`Pages/Settings.tsx`** — بيستخدم `settings` table
